🎯 RINGKASAN 4 LANGKAH MUDAH:

┌─────────────────────────────────────────────────────────┐
│ LANGKAH 1: BUAT TEMPAT DI GITHUB                        │
└─────────────────────────────────────────────────────────┘

1. Buka browser → https://github.com/new
2. Isi nama: billing-system
3. Pilih: PRIVATE ⚠️
4. Klik: Create repository
5. Copy URL yang muncul (simpan dulu)

═══════════════════════════════════════════════════════════

┌─────────────────────────────────────────────────────────┐
│ LANGKAH 2: BERSIHKAN FILE (OPSIONAL)                    │
└─────────────────────────────────────────────────────────┘

Di Cursor:
1. Tekan: Ctrl+` (buka terminal)
2. Ketik: .\cleanup-before-git.bat
3. Enter

Atau skip langkah ini (tidak masalah)

═══════════════════════════════════════════════════════════

┌─────────────────────────────────────────────────────────┐
│ LANGKAH 3: UPLOAD PAKAI CURSOR (INI YANG PENTING!)      │
└─────────────────────────────────────────────────────────┘

A. Buka Source Control:
   - Klik icon cabang pohon di sidebar kiri
   - Atau tekan: Ctrl+Shift+G

B. Initialize (kalau diminta):
   - Klik: "Initialize Repository"

C. Cek File (PENTING!):
   ✅ HARUS ADA: src/, views/, package.json
   ❌ JANGAN ADA: .env, node_modules/, backups/
   
   ⚠️ KALAU .env MUNCUL = STOP! BAHAYA!

D. Stage Files:
   - Klik tombol "+" di samping "Changes"

E. Commit:
   - Ketik message: "Initial commit - v1.0.0"
   - Klik tombol ✓ atau Ctrl+Enter

F. Add Remote (sambung ke GitHub):
   - Klik "..." (3 titik) di pojok kanan atas
   - Pilih: Remote → Add Remote
   - Paste URL GitHub (dari Langkah 1)
   - Enter

G. Push (Upload):
   - Klik "..." lagi
   - Pilih: Push
   - Tunggu selesai

═══════════════════════════════════════════════════════════

┌─────────────────────────────────────────────────────────┐
│ LANGKAH 4: CEK BERHASIL ATAU TIDAK                      │
└─────────────────────────────────────────────────────────┘

1. Buka browser
2. Pergi ke URL GitHub repository Anda
3. Refresh
4. Cek: Apakah file sudah muncul?
   → Kalau ada = BERHASIL! 🎉
   → Kalau kosong = Ada masalah (lapor ke saya)

═══════════════════════════════════════════════════════════

🆘 TROUBLESHOOTING CEPAT:

• .env muncul di file list
  → STOP! Jangan lanjut! Kasih tahu saya!

• "Failed to push"
  → Cek koneksi internet
  → Cek URL GitHub benar atau tidak

• Diminta password
  → Pakai Personal Access Token (bukan password biasa)
  → Bikin di: https://github.com/settings/tokens

• Masih bingung
  → Baca: PANDUAN_MUDAH_GIT.md
  → Atau kasih tahu saya stuck di mana

📌 TOMBOL PENTING DI CURSOR:

Ctrl+Shift+G  = Buka Source Control
Ctrl+`        = Buka Terminal
Ctrl+Enter    = Commit (setelah ketik message)

Di Source Control panel:
"+"           = Stage files
"✓"           = Commit
"..."         = Menu lainnya (Remote, Push, Tags, dll)
